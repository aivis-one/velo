## Раздел 9 — Мелкие улучшения и polish

### SUGGESTION 1 — «Подробнее» выглядит как ссылка, но не кликабельна

`frontend/src/views/user/UserDashboardView.vue` — блок `.dashboard__ai-more`

Элемент содержит стрелку (`IconArrowRight`) и текст «Подробнее» — визуальная affordance ссылки. `cursor` не задан (наследует дефолт стрелки). Пользователь Telegram Mini App, наводя палец, не получает никакой обратной связи, что элемент временно заблокирован.

Пока маршрут `user-ai-summary` не существует — явно выставить `cursor: default` и `opacity: 0.5`, чтобы визуально передать «заглушка».

```css
/* Временно, до wire-up экрана 16 */
.dashboard__ai-more {
  cursor: default;
  opacity: 0.5;
}
```

После появления маршрута: убрать оба свойства, добавить `@click` → `router.push({ name: 'user-ai-summary' })` и `cursor: pointer`.

Риск: низкий (UX, не баг).

### SUGGESTION 3 — Формат строки `checkinAlertTime` изменился без сепаратора

`frontend/src/views/user/UserDashboardView.vue:289-294`

Было: `"Название · через 30 мин"` → Стало: `"Название через 30 мин"`.  
При длинном названии практики две части сливаются: `«Утренняя йога с Павлом через 10 мин»`.

Вариант: вернуть тихий разделитель (не `·`, а `•` уже используется в feedback-баннере):

```typescript
if (diffMinutes < 60) return ` • через ${diffMinutes} мин`
return ` • через ${hours} ч`
```

Риск: только читаемость, на функционал не влияет.

## Раздел 11 — Кросс-модульная согласованность

**Иконки настроения vs. градиентный id.** Все три компонента (`IconMoodLow`, `IconMoodMid`, `IconMoodHigh`) используют `useId()` для уникального `<linearGradient id>`. Паттерн согласован, коллизий нет.

**`VButton variant="secondary"` vs `"outline"`.** Кнопка Zoom переключена с `outline` на `secondary`. В `VButton.vue` `outline` — алиас для `secondary`, изменение безопасно.

**`velo-bg-card-solid`** объявлена в `variables.css`, используется в трёх местах дашборда — согласованно.

**`useId()` требует Vue 3.5+.** `package.json` фиксирует `"vue": "^3.5.0"` — требование выполнено.
