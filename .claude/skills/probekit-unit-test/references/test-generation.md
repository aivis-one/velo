# Test Generation Rules

These rules apply to every test generated by the unit-test skill.
Read before generating any test code.

---

## Core Structure: AAA

Every test follows Arrange / Act / Assert. Always annotate with comments
in the first tests of each file to establish the pattern for the reader.

```python
import pytest

def test_calculate_discount_for_vip_customer_returns_20_percent():
    # Arrange
    customer = Customer(tier="VIP", account_age_days=365)
    cart = Cart(items=[Item(price=100.0)])

    # Act
    discount = calculate_discount(customer, cart)

    # Assert
    assert discount == pytest.approx(20.0)
```

Never merge all three phases into one line. Never skip Assert.
Never assert only that no exception was raised — assert the actual result.

---

## Naming Convention

Pattern: `test_{function_or_method}_{scenario}_{expected_outcome}`

Good:
- `test_parse_date_with_empty_string_raises_value_error`
- `test_add_item_to_cart_updates_total_price`
- `test_user_login_with_wrong_password_returns_401`

Bad:
- `test_1`, `test_login`, `test_parse`, `test_it_works`

---

## Coverage Targets Per Function

For each public function/method, generate minimum:

| Scenario | Mandatory |
|----------|-----------|
| Happy path (valid input, expected output) | ✅ always |
| Empty input (empty string, empty list, None/null) | ✅ always |
| Boundary value (zero, negative, max int, very long string) | ✅ always |
| Expected exception (invalid type, out of range) | ✅ if function raises |
| Async behavior | ✅ if function is async |
| Concurrent access | if function uses shared state |

Skip: private methods (test via public API), trivial getters/setters (no logic).

## Parametrize for Edge Cases

When the same logic runs on multiple inputs — use parametrize, not repeated tests.

```python
@pytest.mark.parametrize("quantity, expected_error", [
    (0,    "Quantity must be positive"),
    (-1,   "Quantity must be positive"),
    (None, "Quantity cannot be None"),
    ("a",  "Quantity must be an integer"),
])
def test_create_order_with_invalid_quantity_raises_value_error(quantity, expected_error):
    with pytest.raises(ValueError, match=expected_error):
        create_order(product_id=1, quantity=quantity)
```

## Property-Based Tests (for pure functions)

For pure functions with mathematical properties, generate property-based tests
alongside example-based tests using hypothesis (Python) or fast-check (JS/TS).

See `references/property-based-testing.md` for full property catalog and tool usage.

**When to generate PBT:**
- Function is pure (no side effects, no I/O, no state mutation)
- Function has a testable property: roundtrip, idempotency, commutativity, invariant preservation
- Example: `encode`/`decode`, `sort`, `normalize`, `format`/`parse`, arithmetic

**When NOT to generate PBT:**
- Function requires complex mock setup
- Function has very few valid inputs (enum-like)
- Function has side effects (DB, HTTP, file I/O)

```python
from hypothesis import given, strategies as st

# Alongside example-based tests for specific edge cases:
def test_encode_empty_string():
    assert encode("") == ""

# Add PBT for the roundtrip property:
@given(st.text())
def test_encode_decode_roundtrip(s):
    assert decode(encode(s)) == s
```

PBT tests are optional additions — never replace example-based edge case tests.

## Fixtures

Use fixtures for any setup that is shared across 2+ tests.
Choose scope based on cost:

| Scope | When to use |
|-------|-------------|
| `function` (default) | Mutable state, must be fresh per test |
| `class` | Shared within one test class, reset between classes |
| `module` | Read-only shared data, expensive to create |
| `session` | Config, constants, one-time expensive resource |

```python
@pytest.fixture
def sample_user():
    """Fresh user instance for each test."""
    return User(name="Alice", email="alice@example.com")

@pytest.fixture(scope="module")
def db_connection():
    """Single in-memory DB connection for entire test module."""
    conn = sqlite3.connect(":memory:")
    create_schema(conn)
    yield conn
    conn.close()
```

Place shared fixtures in `conftest.py` at the appropriate directory level.
Never duplicate fixture definitions across test files.

## Mocking Strategy

Mock at the boundary, not inside your code.

**What to always mock:**
- HTTP requests (`requests.get`, `httpx.get`, `aiohttp`)
- Database calls (use in-memory SQLite or mock the connection)
- File system I/O (use `tmp_path` fixture or `unittest.mock.patch`)
- `datetime.now()`, `time.time()`, `random` (non-determinism)
- External services (payment gateways, email senders, cloud APIs)

**What NOT to mock:**
- Business logic classes you own
- Pure functions in the same module
- Standard library data structures

```python
from unittest.mock import patch, MagicMock

def test_fetch_user_profile_returns_parsed_data(mocker):
    # Arrange
    mock_response = MagicMock()
    mock_response.json.return_value = {"id": 1, "name": "Alice"}
    mock_response.status_code = 200

    # Act
    with patch("myapp.services.requests.get", return_value=mock_response):
        result = fetch_user_profile(user_id=1)

    # Assert
    assert result.name == "Alice"
    assert result.id == 1
```

For async code use `AsyncMock`:

```python
from unittest.mock import AsyncMock, patch

@pytest.mark.asyncio
async def test_async_fetch_returns_data():
    mock_get = AsyncMock(return_value={"status": "ok"})
    with patch("myapp.client.http_get", mock_get):
        result = await fetch_status()
    assert result["status"] == "ok"
```

## Testing Exceptions

Always assert the exception TYPE and MESSAGE, not just that something was raised.

```python
# Good — checks type and message
def test_withdraw_more_than_balance_raises_insufficient_funds():
    account = BankAccount(balance=100)
    with pytest.raises(InsufficientFundsError, match="Available: 100"):
        account.withdraw(200)

# Bad — too permissive
def test_withdraw_raises():
    with pytest.raises(Exception):
        account.withdraw(200)
```

## Testing Return Values: Equality and Approx

- Use `==` for exact values (integers, strings, booleans, lists of known values)
- Use `pytest.approx()` for floats
- Use `assert x is None` / `assert x is not None` — never `assert x == None`
- For collections: assert length + key members, not the entire object if it's large

---

## Isolation Rules

Each test must be independent. A test must not:
- Rely on state set by a previous test
- Write to shared mutable globals
- Leave side effects (files, DB rows, env vars) after completion

Use `yield` fixtures for teardown. Use `tmp_path` for file operations.

## Test File Organization

```
tests/
  conftest.py              # Shared fixtures for entire test suite
  unit/
    test_user_service.py   # Tests for src/services/user_service.py
    test_cart.py
  integration/             # Separate dir — do not mix with unit tests
```

Group related tests in a class when they share a fixture or context:

```python
class TestUserRegistration:
    def test_valid_email_creates_user(self, db): ...
    def test_duplicate_email_raises_error(self, db): ...
    def test_weak_password_raises_error(self): ...
```

## SQLite-Specific Patterns

For code using SQLite (common in Python backends):

```python
@pytest.fixture
def in_memory_db():
    """Isolated in-memory SQLite for each test."""
    conn = sqlite3.connect(":memory:")
    conn.row_factory = sqlite3.Row
    # Apply same schema as production
    with open("schema.sql") as f:
        conn.executescript(f.read())
    yield conn
    conn.close()
```

Never test against a real file-based DB. Always use `:memory:` or a temp file
cleaned up in fixture teardown.

If source code uses `run_in_executor` or `aiosqlite` for async SQLite —
mock the DB layer entirely rather than replicating the async wrapper in tests.

## Pydantic Model Testing

```python
def test_model_rejects_extra_fields_when_forbid_set():
    with pytest.raises(ValidationError):
        StrictModel(**{"expected_field": "ok", "extra_field": "bad"})

def test_model_accepts_none_for_optional_field():
    # Use 'is not None' check, not truthiness — empty dict {} is falsy
    m = MyModel(optional_field=None)
    assert m.optional_field is None

def test_model_dump_roundtrip_preserves_data():
    original = MyModel(name="Alice", value=42)
    restored = MyModel(**original.model_dump())
    assert restored == original
```
