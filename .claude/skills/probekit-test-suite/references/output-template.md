# AUDIT-REPORT Template

Single consolidated document. No intermediate per-stage reports. No duplication.

```markdown
# Audit Report — {target} — {date}

**Tool:** probekit-test-suite v3.0.0 {mode} | **Score:** X.X/10 | **Gate:** PASS/WARN/FAIL
**Scope:** N .py + N .gd | **Tests:** N pass / N fail / N% coverage

---

## 1. Fixed During Audit (N fixes, verified by test suite)

| # | File | Problem | Fix |
|---|------|---------|-----|
| F1 | `path/file.py:line` | Description of the bug | What was changed |

Test patch notes (if mock targets changed due to fixes).

---

## 2. Remaining — P1 (Important, next sprint)

| # | File | Problem | Effort |
|---|------|---------|--------|
| R1 | `path/file.py:line` | Description | Time estimate |

## 3. Remaining — P2 (Medium, backlog)

| # | Area | Problem | Effort |
|---|------|---------|--------|
| R5 | `area` | Description | Time estimate |

## 4. Remaining — P3 (Recommended, future)

| # | Area | Recommendation |
|---|------|---------------|
| R11 | `area` | Description |

---

## 5. Architecture Strengths (DIAMOND patterns)

1. **Pattern Name** — one-line description
...top 10 patterns across all stages

---

## 6. Test Health

| Suite | Tests | Pass | Coverage |
|-------|-------|------|----------|
| Unit | N | N pass, N skip | N% |
| Integration | N | N pass | — |
| Total | N | N pass | N% |

Warnings summary (non-critical only).

---

## 7. Runtime Health

- **Databases:** status
- **Logs:** status
- **Services:** status
- **API Sync:** status

---

*Generated by probekit-test-suite v3.0.0 {mode} mode, {date}*
```

Save to `{{report_dir}}/AUDIT-REPORT-{YYYYMMDD}.md`
After saving, output brief summary in chat.

**Rules:**
- ONE report per audit run. Delete any intermediate SUITE-REPORT files.
- No duplication between sections. Each finding appears exactly once.
- P1/P2/P3 classification based on impact and effort, not severity count.
- Fixed section includes ALL fixes (both auto-fix and manual mid-pipeline).
- Skipped stages appear in Pipeline Summary with reason, not in findings.

