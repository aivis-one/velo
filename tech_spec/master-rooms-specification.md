## Executive Summary

**YON Master Rooms** — встроенная инфраструктура для мастеров и фасилитаторов внутри экосистемы YON, где живые практики, группы и индивидуальная работа соединяются с AI-аналитикой и данными о состоянии пользователей.

### Ключевая ценность

Рабочее пространство мастера и точка входа пользователей в экосистему YON, где мастера могут не только проводить практики, собирать группы, управлять оплатами и контентом, а пользователи — легко подключаться к живым и записанным сессиям в одном приложении.

### Формула продукта

```
YON Master Rooms = 
  живые групповые практики 
  + удобное подключение 
  + возможности для оплаты 
  + комьюнити 
  + AI-рефлексия после практики
```

**Всё — в одном приложении, без выхода во внешние сервисы.**

## Problem Statement

### Pain Points мастеров

**Текущая ситуация:** Разрозненные инструменты для проведения практики

1. **Разрозненные инструменты** — мастера используют Zoom, Telegram, Spotify для практик, но всё это не связано
2. **Потери дохода** — постоплата и забывшие участники не возвращаются
3. **Операционная перегрузка** — напоминания, ссылки, синхронизация требуют времени
4. **Сложность масштабирования** — рост аудитории требует больше ручной работы
5. **Отсутствие понимания запросов** — мастер не знает уровень участников перед практикой
6. **Нет инструментов адаптации** — невозможно получить фидбэк от каждого участника практики для будущих занятий

### Pain Points пользователей

**Текущая ситуация:** Сложный вход и отсутствие интеграции

1. **Нет единого пространства** — каждая практика в своём месте (Zoom/Telegram/другое)
2. **Сложный вход** — чаты, ссылки, оплаты разбросаны
3. **Нет связи с рефлексией** — опыт практики не интегрируется с отслеживанием состояния
4. **Сложно отследить прогресс** — как меняется состояние из практики в практику

## Core Features

### Что включает Master Rooms

#### 1. Персональное пространство мастера внутри YON

- **Единый аккаунт мастера** (без дублирования пространства)
- **Профиль мастера:**
  - Подход / метод
  - Формат работы (группы / 1:1)
  - Опыт, сертификации (опционально)
- **Статус доступности** (принимаю новых участников / пауза)

#### 2. Живые регулярные практики, 1:1 и записи

- **Форматы практик:**
  - Live-групповая практика
  - Регулярная практика (серия)
  - Индивидуальная сессия (1:1)
  - Replay (записи)

#### 3. Автоматизация операционной деятельности

- Оплаты
- Доступы
- Напоминания

#### 4. Данные о состоянии группы до и после практики

- **Интеграция с YON State Engine** — мастер видит состояние группы в реальном времени
- **Динамика после практики** — как изменились состояния участников

#### 5. Возможность для пользователя отслеживать состояние после практики

- **Личный трекинг** — пользователь видит динамику до/после
- **AI-рефлексия** — персонализированная обратная связь

#### 6. Узнавать о новых подходах

- **Discovery механика** — пользователи находят новых мастеров
- **Рекомендации от AI** — подбор практик под текущее состояние

## Detailed Functionality

### 1. Кабинет мастера (Core)

#### Единый аккаунт мастера (без дублирования пространства)

**Профиль мастера включает:**

- **Подход / метод** — описание уникального стиля работы
- **Формат работы** — группы / 1:1 сессии
- **Опыт, сертификации** (опционально) — доверие через прозрачность

#### Статус доступности

- **Принимаю новых участников / пауза** — мастер управляет потоком клиентов
- **Сигналы риска** (без персональных данных) — система помогает мастеру видеть тревожные паттерны в группе

---

### 2. Создание и управление практиками

#### Типы практик

1. **Live-групповая практика** — синхронная сессия в реальном времени
2. **Регулярная практика (серия)** — повторяющиеся занятия (например, еженедельная медитация)
3. **Индивидуальная сессия (1:1)** — персональная работа с клиентом
4. **Replay (записи)** — доступ к архиву прошедших практик

#### Настройки практики

- **Дата / время / длительность** — гибкое планирование
- **Лимит участников** — контроль размера группы
- **Формат:**
  - Тихо
  - С голосом
  - С музыкой
- **Уровень:**
  - Начальный
  - Продвинутый
- **Теги состояний:**
  - Тревога
  - Фокус
  - Заземление
  - И т.д.

**Интерпретация:** Мастер получает гибкий конструктор для создания любых типов практик — от одноразовых ивентов до регулярных серий с повторяющимся расписанием.

---

### 3. Доступы и оплаты (без ручной модерации)

#### Автоматизация платежей

- **Автоматическая проверка оплаты** — система сама валидирует доступ
- **Авто-доступ в практику:**
  - После оплаты за 24 часа
  - За 1 час
  - За 10 минут (напоминания)
- **История посещений и оплат** — полная прозрачность для мастера

#### MVP-допущение

**Возможность подключать внешние платёжные ссылки** (например, Stripe/PayPal) для обработки оплат до создания встроенной биллинговой системы.

**Интерпретация:** На MVP-стадии мастер может использовать любую платёжную систему, но YON автоматизирует доступы после подтверждения оплаты.

---

### 4. Работа с группой (ключевой дифференциатор)

#### Check-in перед практикой

**Для пользователя (быстрый опрос):**

- **Короткий вопрос:** "Как ты себя чувствуешь сейчас?"
- **Выбор состояния** (эмоциональная шкала) — визуальный слайдер или набор эмоджи
- **Опциональный текстовый запрос** — пользователь может уточнить запрос

**Для мастера (агрегированно):**

- **Среднее состояние группы** — общая картина перед началом
- **Распределение по уровням** — сколько новичков / опытных участников
- **Сигналы риска** (без персональных данных) — анонимные индикаторы тревожных состояний

**Интерпретация:** Мастер получает "срез" группы перед практикой, что позволяет адаптировать подход в реальном времени. Пользователи чувствуют персонализацию.

---

### 5. Проведение практики

#### Встроенная видеосессия

- **Минимальный интерфейс** (без чатов и отвлечений) — фокус на практике
- **Опциональный таймер** / музыка / тишина
- **Режим "присутствия"** (камера по желанию) — гибкость для участников

**Интерпретация:** Платформа не пытается быть "ещё одним Zoom" — это пространство для осознанности, где интерфейс минимален и не отвлекает.

---

### 6. После практики (удержание и ценность)

#### Для пользователя

**Короткая рефлексия:**

- Мгновенная обратная связь после практики
- AI-анализ изменения состояния (до → после)
- Рекомендации от AI:
  - **Продолжить** — если практика подошла
  - **Отдохнуть** — если нужна пауза
  - **Другая практика** — если есть более подходящий формат

**Для мастера:**

- **Автофидбэк по группе** — агрегированная статистика изменений состояний
- **Динамика состояний** (до / после) — визуализация эффективности практики
- **Анонимные комментарии** — обратная связь без привязки к конкретным людям

**Интерпретация:** Рефлексия превращает разовую практику в долгосрочный опыт. AI помогает пользователю понять, что работает, а мастеру — как улучшить подход.

---

### 7. Библиотека и повторное использование

#### Для мастера

- **Сохранение записей практик** — автоматический архив
- **Теги по состояниям и запросам** — структурированная библиотека
- **Поиск по библиотеке** — быстрый доступ к нужным записям
- **Доступ для пользователей с подпиской** — дополнительный revenue stream

**Интерпретация:** Мастер не теряет контент — каждая практика становится активом, который можно монетизировать через подписку или продавать отдельно.

---

### 8. Апсейлы внутри MVP

#### Для мастера

- **Запрос на индивидуальную сессию** после практики — тёплый лид
- **Подписка на регулярные практики мастера** — recurring revenue
- **Рекомендации других мастеров** внутри YON — кросс-промоушн

**Интерпретация:** Платформа активно помогает мастеру расти — от групповых практик к 1:1 сессиям, от разовых к регулярным подпискам.

## Business Model

### Архитектура платформы

**YON = мультисторонняя платформа + AI-инфраструктура состояний**

Платформа соединяет пользователей, мастеров / фасилитаторов, AI-ядро YON, извлекая ценность не только из транзакций, но и из данных об эффективности практик и состояниях людей.

---

### Three-Sided Platform

#### Сторона 1: Пользователи (Demand Side)

**Что получают:**

- **Единый доступ** к практикам разных мастеров
- **Живые сессии + записи** — гибкость выбора формата
- **Персональные рекомендации** под текущее состояние
- **Дневник, рефлексия, AI-навигация** — интеграция с YON State Engine

**Поведенческая модель:**

- Пользователь **не привязывается к одному мастеру** — выбирает практику под состояние здесь и сейчас
- Со временем **повышает частоту использования** — регулярность через подписку

**Ценность для бизнеса:**

- **Высокий retention** за счёт эмоциональной пользы
- **Рост LTV** через подписку
- **Данные о состоянии** и паттернах восстановления — основа для улучшения рекомендаций

---

#### Сторона 2: Мастера / фасилитаторы (Supply Side)

**Что получают:**

- **Готовую инфраструктуру** для практик — не нужно собирать tech stack самостоятельно
- **Автоматизацию операционки** — оплаты, доступы, напоминания
- **Выход к новой аудитории** без маркетинговых затрат — платформа приводит клиентов
- **Аналитику по группе** и динамике состояний — data-driven подход к практике

**Почему остаются:**

- **Отсутствие рутинных процессов** — больше времени на саму практику
- **Снижение операционной нагрузки** — автоматизация высвобождает ресурсы
- **Рост дохода** через:
  - Регулярные группы
  - 1:1 после практик
  - Привлечение новых клиентов и повышение репутации

**Ценность для бизнеса:**

- **Мастера = канал привлечения пользователей** — каждый мастер приводит свою аудиторию
- **Низкий CAC** — органический рост через сарафан
- **Рост supply → рост value** платформы — больше мастеров = больше разнообразия для пользователей

---

#### Сторона 3: AI-ядро YON (Value Engine)

**Функции AI:**

- **Анализ чек-инов** до практик — понимание текущих состояний
- **Анализ фидбэка** после практик — оценка эффективности
- **Навигация состояний** — персональные рекомендации практик

**Для кого:**

- **Для пользователей:** персонализация (какая практика подходит сейчас)
- **Для мастеров:** инсайты (какие состояния группа приносит на практику)

**Роль в бизнес-модели:**

- **Персонализация рекомендаций** — увеличение конверсии в оплату
- **Повышение эффективности** практик — рост retention
- **Формирование data moat** — чем больше данных, тем точнее рекомендации

**AI становится не ассистентом, а навигатором состояний.**

---

### Value Flows (Потоки ценности)

#### Поток 1: Пользователь → YON

**Пользователь платит:**

- За **доступ к практикам** (разовая оплата или подписка)
- За **контент** (записи практик)

**Пользователь получает:**

- Единый доступ к экосистеме мастеров
- Персональные рекомендации под состояние
- AI-рефлексия и навигация

---

#### Поток 2: Мастер → YON

**Мастер получает:**

- **Доход** от практик (платформа берёт комиссию или подписку)
- **Инфраструктуру** — не нужно строить tech stack самостоятельно
- **Новую аудиторию** — platform-driven growth

**YON удерживает комиссию** за предоставление инфраструктуры и аудитории.

---

#### Поток 3: YON → Мастер

**YON выплачивает:**

- Комиссию за практики
- Выплаты за 1:1 сессии

**Мастер получает:**

- Автоматизированную операционку (оплаты, доступы, напоминания)
- Данные о группе для улучшения практик
- Новых клиентов через marketplace

---

### Замкнутый цикл роста

```
Больше пользователей 
  → больше данных о состояниях
  → лучшие рекомендации
  → выше retention
  → выше LTV
  → больше ценности для мастеров
  → больше мастеров присоединяются
  → больше разнообразия практик
  → больше пользователей привлекается
```

**Network effect:** Рост одной стороны усиливает ценность для других.

---

### Почему модель масштабируема

#### 1. Не зависит от одного мастера

- Платформа агрегирует мастеров, а не полагается на одного "звёздного" ведущего
- Рост supply = рост разнообразия для пользователей

#### 2. Не упирается в контент-производство

- Мастера создают контент, а не платформа
- YON предоставляет инфраструктуру, а не производит практики сама

#### 3. Растёт вместе с количеством практик и состояний

- Чем больше практик проведено, тем больше данных
- Чем больше данных, тем точнее AI-навигация

#### 4. Применима глобально (low localization cost)

- Практики осознанности универсальны
- Не требуется глубокая локализация (в отличие от образовательных платформ)

---

### Долгосрочный потенциал

#### 1. Стандарты эффективности практик

- **AI-рейтинг** форматов и подходов — какие практики работают лучше
- **Benchmark** для мастеров — сравнение с лучшими

#### 2. B2B-направления (корпоративный wellbeing)

- Закрытые практики для команд
- Корпоративные подписки
- Аналитика состояния сотрудников (агрегированно, анонимно)

## Monetization

### Принцип монетизации YON

**Ключевая идея:**

- **Пользователь платит за доступ к состояниям и навигации**, а не на маркетинг
- **Платформа зарабатывает многократно** на одном пользователе через разные потоки

**Монетизация встроена нативно** и не нарушает ощущение доверия и безопасности.

---

### Revenue Streams

#### Поток 1: Membership-подписка (Core Revenue Stream)

**Что входит:**

- Доступ к **live-практикам** разных мастеров
- **Библиотека записей** — архив всех прошедших практик
- **AI-рекомендации** практик под текущее состояние
- **Дневник и рефлексия** — интеграция с YON State Engine

**Почему работает:**

- Пользователи **не хотят быть привязаны к одному мастеру** — ценность в разнообразии и регулярности
- **Ценность — в subscription**, а не в разовых покупках

**Доход платформы:**

- **Фиксированный регулярный MRR** — предсказуемый cash flow
- **Высокий LTV** — чем дольше пользователь остаётся, тем выше lifetime value

---

#### Поток 2: Marketplace-комиссия % с платных практик / ивентов

**Что монетизируется:**

- **Специальные практики** — уникальные форматы от топовых мастеров
- **Коллаборации мастеров** — совместные практики нескольких ведущих
- **Тематические события** вне подписки — разовые ивенты (например, ретрит выходного дня)

**Механика:**

- Пользователь **платит за конкретную практику**
- **YON удерживает комиссию** (например, 20-30% от стоимости)

**Доход платформы:**

- **% с транзакций** — чем больше практик проводится, тем больше доход
- **Рост ARPU** без повышения базовой подписки — дополнительный revenue на тех же пользователях

---

#### Поток 3: Индивидуальные сессии 1:1 (High-Margin Stream)

**Как возникает:**

- После групповой практики пользователь хочет **персональную работу с мастером**
- По **рекомендации мастера или AI** — система предлагает 1:1 формат

**Почему это важно:**

- **Высокая ценность** для пользователя — глубокая персональная работа
- **Высокий чек** — индивидуальные сессии стоят дороже групповых
- **Минимальный CAC** — лид уже тёплый после групповой практики

**Доход платформы:**

- **Комиссия с каждой 1:1-сессии** — процент от стоимости
- **High-margin** поток без дополнительных маркетинговых затрат

---

#### Поток 4: Подписка / Pro-инструменты для мастеров

**MVP: бесплатно (стратегия привлечения supply)**

**Phase 2 (после роста базы мастеров):**

- **Расширенная аналитика** по группам — детальные инсайты о динамике состояний
- **Приоритет в рекомендациях** — мастера с Pro-аккаунтом показываются выше
- **Расширенная библиотека записей** — больше хранилища для архива практик

**Доход платформы:**

- **B2B-подписка** от мастеров
- **Предсказуемый доход** (recurring revenue от supply side)

---

#### Поток 5: Корпоративные и закрытые практики (Phase 2)

**Формат:**

- **Закрытые комнаты** для команд / сообществ
- **Регулярные wellbeing-практики** для сотрудников компаний

**Доход платформы:**

- **Корпоративные контракты** — долгосрочные B2B-сделки
- **Высокий чек** — компании платят за wellbeing сотрудников значительно больше, чем индивидуальные пользователи

---

### Сводная схема потоков доходов

```
Пользователь → YON:
  - Подписка (recurring)
  - Оплата спец. практик (transactional)
  - Оплата 1:1 (high-margin)

YON → Мастер:
  - Выплаты за практики (комиссия удержана)
  - Выплаты за 1:1 (комиссия удержана)

Мастер → YON (Phase 2):
  - Комиссия с практик (если мастер ведёт платные)
  - Pro-подписка (если мастер хочет расширенные инструменты)
```

---

### Почему модель устойчива

#### 1. Диверсифицированные потоки доходов

- Платформа **не зависит от одного сегмента** (B2C/B2B/Marketplace)
- Если один поток падает, другие компенсируют

#### 2. Нет зависимости от одного мастера или сегмента

- Рост supply → рост разнообразия → рост пользовательской базы
- Уход одного мастера не обрушивает платформу

#### 3. Масштабируется вместе с ростом базы мастеров и пользователей

- Больше пользователей → больше подписок
- Больше мастеров → больше платных практик → больше комиссий
- Больше данных → точнее AI → выше retention → выше LTV

## Appendix

### Terminology

- **YON State Engine** — AI-система анализа эмоциональных и ментальных состояний пользователей (контекст из основной экосистемы YON)
- **Check-in** — быстрый опрос состояния пользователя перед практикой
- **Check-out** — рефлексия после практики
- **Master / Фасилитатор** — ведущий практики (медитация, дыхательные техники, йога, коучинг и т.д.)
- **Live-практика** — синхронная групповая сессия в реальном времени
- **Replay** — запись прошедшей практики, доступная для просмотра
- **Supply Side** — сторона предложения (мастера)
- **Demand Side** — сторона спроса (пользователи)
- **Value Engine** — AI-ядро платформы, создающее ценность через персонализацию
